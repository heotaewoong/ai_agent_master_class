━━━━━━━━━━━━━━━━━━━━━━━━

📌 한 줄 소개
"유튜브 구독 채널 + AI 뉴스 RSS를 한 번에 수집하여, 내 관심 주제에 맞는 개인 맞춤 뉴스레터를 자동으로 만들어주는 LangGraph 에이전트"

정보 홍수 속에서 매일 중요한 AI/테크 뉴스를 일일이 찾아보는 것이 너무 비효율적이라는 문제를 해결하고 싶었어요.


━━━━━━━━━━━━━━━━━━━━━━━━
⚙️ 핵심 기능 3가지
━━━━━━━━━━━━━━━━━━━━━━━━

① 의도 분석 (Intake Node)
→ 어떤 기능인가요?
사용자가 "AI 최신 트렌드 알려줘" 라고 입력하면, LLM이 의도를 분석해서
유튜브만 볼지 / 뉴스만 볼지 / 둘 다 볼지 / 뉴스레터만 쓸지를 자동 판별합니다.

→ 왜 추가했나요?
매번 수동으로 소스를 고르는 게 불편하고, 자연어 한 마디로 에이전트 흐름을
전환할 수 있어야 진짜 에이전트라고 느꼈기 때문입니다.

② 멀티 소스 병렬 수집 (Send API)
→ 어떤 기능인가요?
intent가 "both"일 때 LangGraph의 Send API를 사용해 유튜브 RSS 수집과
뉴스 RSS 수집을 동시에(병렬로) 실행합니다.
Matt Wolfe, Lex Fridman 등 구독 채널 + AI타임스, TechCrunch 등 16개 RSS를 한꺼번에 긁어옵니다.

→ 왜 추가했나요?
순차 실행하면 너무 느려서, LangGraph의 Send API 팬아웃 패턴을 직접 써보고
병렬화의 효과를 경험해보고 싶었습니다.

③ AI 큐레이션 + 뉴스레터 자동 생성
→ 어떤 기능인가요?
수집된 기사 중 토픽 관련도 점수를 매겨 상위 10개를 선별하고,
마크다운 형식의 뉴스레터를 자동으로 작성 + output/ 폴더에 저장합니다.

→ 왜 추가했나요?
30개 기사를 다 읽기는 너무 많으니, 에이전트가 "편집장" 역할을 해서
핵심만 뽑아주는 경험을 만들고 싶었습니다.


━━━━━━━━━━━━━━━━━━━━━━━━
💬 예상 사용 시나리오
━━━━━━━━━━━━━━━━━━━━━━━━

[사용자 입력]
$ python main.py "이번 주 LLM 최신 동향 뉴스레터 써줘"

[에이전트 동작]
1. Intake: intent=both, topics=["LLM", "최신 동향"] 분석
2. 병렬 수집: YouTube RSS (Yannic Kilcher, AI Explained 등) + 뉴스 RSS (16개 소스) 동시 실행
3. Curator: 수집된 기사 중 LLM 관련 상위 10개 선별 + 트렌드 요약 생성
4. Newsletter Writer: 마크다운 뉴스레터 작성 및 output/newsletter_20260511.md 저장

[결과 예시]
📡 NewsHub Daily — LLM, 최신 동향
2026년 05월 11일 | AI가 큐레이션한 오늘의 뉴스

🌟 핵심 트렌드: xAI Grok 4.3 공개, Anthropic 새 모델 출시...
📰 주요 기사 10선 (관련도 순 정렬)
🔮 전망: LLM 시장은 수익성 경쟁으로 전환 중...
